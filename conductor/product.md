# Product Definition

## Goals
- To provide a "Trusted External System" that captures 100% of the user's "open loops," allowing for stress-free productivity through the 5 steps of GTD: Capture, Clarify, Organize, Reflect, and Engage.

## Core Pillars
- **Ubiquitous Capture:** The app must allow for "frictionless" entry of ideas. If it takes more than 5 seconds to start a "Capture," the system fails.
- **Outcome-Oriented Clarifying:** The system must force the user to define a "Successful Outcome" (the Project) and a "Physical Next Action" (the Task).
- **Context-Based Organization:** Tasks should not just be "to-dos" but must be filtered by Context (e.g., @computer, @calls, @home) and Energy Level.
- **Perspective via Horizons of Focus:** The app must link daily chores to higher-level "Roles" and "Ambitions" so the user can see why they are doing a task.
- **Structured Reflection:** Built-in triggers for a Daily Check-in and a Weekly Review to keep the system current and trustworthy.

## Core Features
- **Inbox / Capture Tool:** High-speed entry point for all open loops.
- **Project Manager:** Define successful outcomes and link them to next actions.
- **Contextual Task Lists:** Filterable views based on location, tools, and energy.
- **Horizons View (Hierarchical Organization):**
  - **Horizon 5 (Purpose):** A dedicated "North Star" view for life mission and core values.
  - **Horizon 4 (Vision):** 3-5 year goal tracking.
  - **Horizon 2 (Areas of Focus / Roles):** A system to categorize every project under a specific "Role" (e.g., Parent, Developer, Athlete).
- **Review Module:** Dedicated interfaces for Daily and Weekly reviews.
  - **Constraint:** If a role has no active projects, the system should flag it during the Weekly Review to ensure all life areas are moving forward.

## External Integrations (Contextual Capture)
- **Todoist Sync:** Bi-directional sync where Todoist acts as the "Mobile Capture" and "Execution" tool, while this app acts as the "Brain" and "Planning" tool.

## User Experience (UX)
- **Frictionless:** Minimal taps/clicks to get information in.
- **Trustworthy:** Consistent reminders and structure to prevent loops from being lost.
- **Focused:** Dynamic filtering to show only what can be done in the current context.

## Non-Goals (What NOT to build)
- **Not a Calendar:** Use the user's existing calendar for hard-scheduled items; this app is for "Next Actions."
- **Not a Social Tool:** This is a private, personal system. No sharing or collaboration features.
- **No "Due Dates" by Default:** In GTD, dates are only for things that must happen on that day. Most tasks should be "Next Actions," not arbitrary deadlines.
